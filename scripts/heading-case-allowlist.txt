# acronyms, or deliberate stylings that the heuristic can't tell from a
# sentence-case mistake. The canonical rule is brik-bds#910 (typography.mdx

#
# Format: one full, exact heading string per line (matched after trim).
#         Lines starting with `#` are comments.
# Prefer this only for genuine exceptions — most headings should just be
# title-cased. For a one-off, an inline `/* lint-heading-case-ignore */`
# comment on the same line works too.
