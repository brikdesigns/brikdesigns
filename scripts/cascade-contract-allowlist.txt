# scripts/cascade-contract-allowlist.txt
#
# Transitional exemptions for the cascade-contract gate
# (@brikdesigns/bds/cascade-contract-check, ADR-013 §2). Each entry is a token
# whose REDEFINITION in consumer CSS is tolerated for now. Goal: drive to zero.
#
# Format: one bare token name per line (`--` optional). `#` starts a comment.
# A name here exempts redefinitions of that token anywhere in src/**.
#
# ── Heading scale: RESOLVED (brik-bds #928, bds ≥0.99.0) ─────────────────────
# The :root heading remap is retired — brikdesigns adopts the `expressive`
# typography variant (`data-mode-typography="expressive"` on <html>), which emits
# the identical scale from bds-tokens. ADR-013 §3 proof-of-loop closed; the 7
# `--heading-*` entries were removed here (brikdesigns #534).
#
# ── Contextual contrast pins — NOT transitional ─────────────────────────────
# `--text-primary` / `--text-secondary` are rebound on specific surface
# contexts (`.service-surface`, `.lp-surface-solid`, `.lp-form-card` + the
# nested-card exceptions) so BDS components in the subtree inherit a readable
# foreground on fixed-light service tints / solid brand surfaces — the #360 /
# #391 contrast pattern, not a Brand-Kit variation. They can't move to
# `.theme-{client}` scope (surface-context-specific, not brand-wide) and can't
# be removed without breaking contrast. The cascade-contract gate has no
# per-line escape (unlike the token-family rule's `bds-lint-ignore`), so the
# global allowlist is the only suppression available today. Burn-down needs a
# brik-bds gate feature: an inline cascade-contract ignore directive mirroring
# `/* bds-lint-ignore token-family — … */`. Until then these stay
# (see brikdesigns #534 hand-off + ADR-013 §2).
text-primary
text-secondary
